# Release Notes - Helm Monorepo Automation

All notable changes to this project will be documented in this file.

## [1.1.0] - 2026-04-16
### Added
- **Hierarchical Health Checks (v3)**: Support for separate `liveness` and `readiness` configurations with independent timing and paths.
- **Smart Protocol Detection**: Automatically switches between `httpGet` and `tcpSocket` based on the presence of the `path` field in health configuration.
- **Opt-in Safety**: Probes are now only generated if `health.enabled: true` is explicitly set, preventing accidental crash loops.

### Fixed
- Fixed a bug where `null` values for probes were being written to `values.yaml`, which could cause issues with certain Helm/Kubernetes schemas.

---

## [1.0.0] - 2026-04-15
### Added
- **Core Generator**: Python-based engine to scaffold Helm Charts from minimal YAML definitions (`apps.<env>.yaml`).
- **Shared Resources Mechanism**: Automatic generation of project-wide `ConfigMaps` and `Secrets` (`project-shared` chart).
- **Vault Integration**: Bash-based `vault_sync.sh` script to pull secrets from HashiCorp Vault (KV v2) into Kubernetes.
- **GitLab CI/CD Integration**: Full pipeline support for chart generation, secret synchronization, and image tagging.
- **Security Hardening**: Standard application templates now defaults to non-root, read-only filesystems with dropped capabilities.
- **Environment Support**: Centralized variable management via `variables.env` per project.
